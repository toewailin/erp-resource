## ၂။ လိုအပ်ချက် နားထောင်ခြင်းနှင့် မေးခွန်းများ (Listening & Question Phase)

### 👂 Listen (နားထောင်ခြင်း)
* Client ဘက်က လက်ရှိလုပ်ငန်းအခြေအနေနဲ့ လိုအပ်ချက်တွေကို ပြောနေစဉ် သေသေချာချာ နားထောင်ပါ။ (ဖြတ်မပြောရ)

### ❓ Questions for Client (လုပ်ငန်းရှင်သို့ မေးရန်)
> "လက်ရှိ လုပ်ငန်းလည်ပတ်တဲ့ နေရာမှာ ဘယ်အပိုင်းတွေက အခက်လဲဆုံး ဖြစ်နေလဲခင်ဗျာ။ ဒီ System ကို သုံးပြီး ဘယ်လို ပြဿနာတွေကို အဓိက ဖြေရှင်းချင်တာလဲဗျ။"

### ❓ Questions for Accountant (စာရင်းကိုင်သို့ မေးရန်)
> "အစ်မ/အစ်ကို တို့ လက်ရှိ စာရင်းတွေကို ဘယ်လို လုပ်နေလဲ (Excel လား၊ တခြား Software လား)။ Report တွေ ထုတ်တဲ့အခါ ဘယ်လို အချက်အလက်မျိုးတွေကို အများဆုံး ကြည့်ချင်လဲ?"

## ၃။ ဖြေရှင်းချက်တင်ပြခြင်း (Solution Proposal - Phased & Modular Approach)

**ပြောရမည့်ပုံစံ:**

> "ဟုတ်ကဲ့၊ အစ်ကိုတို့ လုပ်ငန်းသဘာဝကို နားလည်ပါပြီ။ ဒါမျိုးဆိုရင် တစ်ခါတည်း အကုန်လုံးကို ရေးတာထက်၊ စောစောက ပြောသွားတဲ့ အရေးအကြီးဆုံး **(____)** နဲ့ **(___)** ပိုင်းကို **Phase 1** အနေနဲ့ အရင်တည်ဆောက်တာ အကောင်းဆုံးပါ။ ပြီးမှ နောက်ထပ်လိုအပ်တဲ့ Features တွေအတွက် **Phase 2** ကို ဆက်ရေးတာ ပိုအဆင်ပြေပါတယ်။"

* **Phase 1:** လုပ်ငန်းလည်ပတ်ဖို့ မရှိမဖြစ် လိုအပ်တဲ့ အခြေခံစနစ် (Core System) နှင့် အရေးကြီး Module များ ပါဝင်မယ်။
* **Phase 2:** နောက်ထပ် လုပ်ငန်းလိုအပ်ချက်ပေါ်မူတည်ပြီး ထပ်မံ တိုးချဲ့ချင်တဲ့ အပိုင်းတွေ ပါဝင်မယ်။

### 💡 Excel Data Management အတွက် ယုံကြည်မှုပေးရန်
> "Data Entry သွင်းရတာ အလုပ်မရှုပ်အောင်၊ လက်ရှိ Excel တွေကနေ Data ဆွဲသွင်းလို့ရအောင် (Import/Export) ကျွန်တော်တို့ PixelVite Team က တာဝန်ယူပြီး သေချာ လုပ်ပေးပါမယ်။"

## ၄။ အချိန်ကာလနှင့် ဈေးနှုန်းဆွေးနွေးခြင်း (Timeline & Pricing Strategy)

⚠️ **သတိပြုရန်:** Client ဘက်က "ဘယ်လောက်ကြာမလဲ၊ ဈေးဘယ်လောက်လဲ" ဟု မေးလာပါက ဈေးနှုန်းကို Meeting တွင် ချက်ချင်း လုံးဝ မဖြတ်ပါနှင့်။ သို့သော် **ဈေးနှုန်း တွက်ချက်မည့် စနစ် (Pricing Model)** ကိုတော့ ရှင်းပြရပါမည်။

**ပြောရမည့်ပုံစံ:**

> "အချိန်ကာလအနေနဲ့ကတော့ **Phase 1 အတွက် [ဥပမာ- ၂ လ/၃ လ] လောက် ကြာနိုင်ပါတယ်**ခင်ဗျာ။ 
> 
> ဈေးနှုန်းနဲ့ ပတ်သက်ပြီး ကျွန်တော်တို့ PixelVite က Tech Agency ကြီးတွေသုံးတဲ့ **'Core System Infra + Module Fee'** စနစ်နဲ့ ပွင့်လင်းမြင်သာစွာ တွက်ချက်ပါတယ်။ 
> 
> * **Core System Infra (အခြေခံစနစ်):** ဒါကတော့ System တစ်ခုလုံးရဲ့ လုံခြုံရေး Authentication၊ Database နဲ့ အခြေခံတည်ဆောက်တာတွေ ဖြစ်ပါတယ်။
> * **Module-Based Pricing:** ကျန်တာတွေကိုတော့ လုပ်ဆောင်ချက် တစ်ခုချင်းစီအတွက် ဈေးဖြတ်ပါတယ်။ ဥပမာ - 'Booking Module တစ်ခု ၂ သိန်း၊ Report Module တစ်ခု ၂ သိန်း' ဆိုပြီး သီးသန့်ခွဲပြပေးမှာပါ။ ERP/Software မျိုး မဟုတ်ဘဲ Website ရေးမယ်ဆိုရင်လည်း Page တစ်မျက်နှာကို ၁ သိန်း ဆိုပြီး တွက်ချက်ပေးတာမျိုးပါ။ ဒီလို လုပ်ပေးတဲ့အတွက် အစ်ကိုတို့ဘက်က ပေးရတဲ့ ငွေကြေးပမာဏနဲ့ ပြန်ရမယ့် လုပ်ဆောင်ချက် (Business Value) ကို ရှင်းရှင်းလင်းလင်း တွက်ကြည့်လို့ လွယ်ကူသွားမှာပါ။
> * **Complex Modules:** အကယ်၍ Payment Gateway (KBZPay), SMS, Google Maps လိုမျိုး ပြင်ပစနစ်တွေ (External APIs) ချိတ်ဆက်ရတဲ့ Complex Module တွေ ပါလာရင်တော့ အချိန်ပိုကြာနိုင်လို့ သီးသန့်ဈေး သတ်မှတ်ပေးပါမယ်။ External API Cost တွေကိုတော့ Client ဘက်က တိုက်ရိုက် ကျခံရပါမယ်ခင်ဗျာ။
>
> ဒီအချက်အလက်တွေပေါ် မူတည်ပြီး **နောက် ၂ ရက်/၃ ရက် အတွင်း** Quotation ကို သေချာတွက်ချက်ပြီး ပေးပို့ပေးပါမယ်ခင်ဗျာ။"

## ၅။ Support နှင့် ရေရှည်ထိန်းသိမ်းမှု (Warranty & Maintenance)

* **Warranty:** စနစ်စတင်အသုံးပြုပြီး **၃ လ အထိ (Warranty) အခမဲ့ Bug fix** ပြုလုပ်ပေးမည်။
* **AMC (Annual Maintenance Contract):** Warranty ကာလ ကုန်ဆုံးသွားပါက တစ်နှစ်စာ ထိန်းသိမ်းခ စာချုပ်ချုပ်ရပါမည်။ (ပုံမှန်အားဖြင့် **Total Project Cost ရဲ့ 15%** သတ်မှတ်ပါသည်)။
    * *အကျုံးဝင်မည့်အရာများ:* System Down လျှင် ချက်ချင်းတက်ဖြေရှင်းပေးခြင်း၊ Server ထိန်းသိမ်းပေးခြင်း၊ လစဉ် Backup စနစ်တကျ ယူပေးခြင်း။
* **Server & Deployment:** ဆာဗာ (VPS) နှင့် ဒိုမိန်းစရိတ်များ။
* **Taxes:** ကုန်သွယ်လုပ်ငန်းခွန် (Commercial Tax) အခွန်အခများ။
